# ✅ JavaScript 冒泡排序（Bubble Sort）+ 经典案例演示

## ✔ 冒泡排序 JS 实现（可直接用）

```js
function bubbleSort(arr) {
  const n = arr.length;
  for (let i = 0; i < n - 1; i++) {
    // 标记这一趟是否发生交换，如果没有交换说明已经有序
    let swapped = false;

    // 每次冒泡都会把当前“最大”的元素移到右侧
    for (let j = 0; j < n - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        // 交换
        [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
        swapped = true;
      }
    }

    // 若没有交换，说明已完全有序，可以提前结束
    if (!swapped) break;
  }
  return arr;
}
```

# 📌 冒泡排序执行过程（经典案例逐步模拟）

数组：`[5, 1, 4, 2, 8]`

---

## 🔹 第一趟（i=0）

依次比较相邻元素，交换让“最大值”往右冒：

- 比较 5 和 1 → 交换 → `[1, 5, 4, 2, 8]`
- 比较 5 和 4 → 交换 → `[1, 4, 5, 2, 8]`
- 比较 5 和 2 → 交换 → `[1, 4, 2, 5, 8]`
- 比较 5 和 8 → 不交换

👉 **8 已经在正确位置（末尾）**

---

## 🔹 第二趟（i=1）

只比较前 4 个：

- 1 和 4 → 无交换 → `[1, 4, 2, 5, 8]`
- 4 和 2 → 交换 → `[1, 2, 4, 5, 8]`
- 4 和 5 → 无交换

👉 **第二大值 5 也到位了**

---

## 🔹 第三趟（i=2）

比较前 3 个：

- 1 和 2 → 无交换
- 2 和 4 → 无交换

👉 已经没有交换了 → 算法提前结束

# ⭐ 冒泡排序特点总结

| 特性         | 描述                             |
| ------------ | -------------------------------- |
| 时间复杂度   | O(n²)，最好情况（已排序）是 O(n) |
| 空间复杂度   | O(1)                             |
| 稳定性       | ✔ 稳定排序                       |
| 是否原地排序 | ✔ 是                             |
| 优点         | 简单、稳定，有“提前结束”优化     |
